# Test Driven Development

Заданието ви тази седмица е да разработите просто приложение, използвайки методологията **Test Driven Development**. **Test Driven Development** e методология за разработване на софтуер, която осигурява качествен код за бързо време.
## Програмата

С цел запознаваме с методологията програмата, която ще разработите е относително проста: **FizBuzz**. Трябва да разработите функция, която да приеме като аргумент цяло положително число и връща:
- "Fizz" ако числото се дели на 3
- "Buzz" ако числото се дели на 5
- "FizzBuzz" ако числото се дели на 3 и на 5
- Самото число, ако горните условия не са изпълнени

## Test Driven Development - Стъпки

При използване на Test Driven Development всяка функционалност на приложението трябва да бъде разработена по следният начин:

1. **Дефиниране на тестов сценарий** - дефинираме какво трябва да върне приложението при определен вход.
2. **Създаваме на тест** - създаваме тест, който да покрие този тестов сценарий. Тестът трябва първоначално да се проваля, понеже функционалността още не е добавена.
3. **Добавяне на функционалност** - променяме кодът на нашето приложение, така че тестът да мине. След като тестът мине, спираме разбработката и да даваме на следващата стъпка.
4. **Рефакториране** - правим кодът по-четлив/чист. След рефакторирането трябва да се уверим че тестовете още минават.

Спазвайки тези стъпки трябва да разработите горното приложение.

### Примерна първа стъпка

### 1. Дефиниране на тестов сценарий

Когато функцията получи число, което не се дели нито на 3, нито на 5, трябва да върне като резултат самото число. Например при вход 1 трябва да върне 1.

### 2. Създаване на тест

```python
# test_fizzbuzz.py

import pytest
from fizzbuzz import fizzbuzz  # Import function (yet to be implemented)

def test_returns_number():
    assert fizzbuzz(1) == 1  # Expect 1 to return 1
```
### 3. Имплементиране на функцията

```python
# fizzbuzz.py

def fizzbuzz(n):
    return n  # Simple return to pass the first test
```

### 4. Рефакториране

```python
# test_fizzbuzz.py

import pytest
from fizzbuzz import fizzbuzz  # Import function (yet to be implemented)

def test_returns_number():
	simple_number = 1 # avoid magic numbers
    assert fizzbuzz(simple_number) == simple_number  # Expect 1 to return 1
```
